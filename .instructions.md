# 《模形式与费马大定理》写作指南

## 项目概述

本书参考 Diamond & Shurman *A First Course in Modular Forms*（GTM 228）的章节顺序，用中文撰写一本从模形式基础到费马大定理 (FLT) 完整证明链的学习笔记。章节结构对应 D&S 的 9 章 + 第 10 章 FLT。

## 互动学习模式

写每个小节之前，先按以下流程与用户互动：

1. **出题摸底**：针对即将写的小节，出 2-3 道题测试用户的前置知识。
2. **用户作答**：等待用户回答。
3. **诊断与补充**：根据回答判断知识缺口，把需要补的背景知识写进书的对应位置（可以是当前章节的 remark/补充小节，或附录）。
4. **讲解核心内容**：确认用户理解前置知识后，讲解本节核心定理/概念。
5. **写入 LaTeX**：将内容（含补充材料）写进 .tex 文件。

用户问"为什么"时，把相关背景知识作为 remark 或补充小节加入书中。

## 用户反馈记录

以下是互动学习中用户提出的具体要求，后续写作必须遵循：

- **记号表放在目录后面**，作为独立的前言页（不在 Chapter 1 内），供查阅用。
- **抽象符号必须配详细解释**：每个抽象代数符号（如 $\SL_2$、$\trianglelefteq$、$\ker$ 等）首次出现时给出直白的中文解释和具体例子，不要光写符号。
- **习题必须带参考答案**：每道练习题后面附上解答（用 `\begin{proof}[解答]...\end{proof}` 环境）。
- **不要一开始堆砌抽象概念**：新概念应在用到时自然引入，不要集中在章节开头罗列。
- 用户不喜欢纯计算题，更关注概念理解和"为什么"。
- **深入浅出，从具体到抽象**：讲新概念（如模形式）时，先给一个具体的例子（如 Eisenstein 级数 $E_4$），让读者看到它长什么样、有什么性质，然后再引出抽象定义。不要先堆一堆抽象定义再举例——那样读者会迷失。正确模式：具体例子 → 观察性质 → 提出问题（为什么有这些对称性？）→ 引入抽象框架。
- **概念在需要时引入，不提前堆砌**：如果某个概念（如群作用、稳定子、轨道、商空间、同余子群等）在当前段落没有立刻用到，就不要在这里定义它。等到真正需要的地方再引入，并配具体例子深入讲解。宁可"重复"也不要"提前"。每次引入概念时问自己：读者此刻为什么需要知道这个？如果回答不出来，就推迟到能回答的地方。
- **术语翻译要自然**：不要用生硬的直译（如"忠实地作用"faithfully acts）。用大白话解释含义，比如把"忠实作用"改成"不同元素对应不同变换"。

## 核心写作原则

### 证明要求
- **每一个定理、命题、引理都必须给出尽可能详细的证明**，不得省略关键步骤。
- 如果某步骤"显然"或"容易验证"，仍需写出完整推导。读者应能仅凭本书完成所有证明。
- 证明中若引用前面的结果，用 `\cref{}` 明确指出来源定理/引理编号。
- 复杂证明应分步骤或分情形(case)展开，使用 enumerate 或 description 环境。

### 数学排版规范
- 定理类环境：`theorem`（定理）、`proposition`（命题）、`lemma`（引理）、`corollary`（推论）、`definition`（定义）、`example`（例）、`remark`（注记）。
- **不同类型环境有不同颜色的左边条**（由 `tcolorboxenvironment` 自动添加）：定理/命题/引理蓝色、定义绿色、例橙色、注记灰色、习题紫色。不需要手动加框，只要用标准 amsthm 语法即可。
- 所有重要的数学概念首次出现时用 `\textbf{}` 加粗标注。
- 使用 `preamble.tex` 中已定义的宏：`\HH`（上半平面）、`\SL`、`\GL`、`\GammaO`、`\GammaI`、`\Mk`（模形式空间）、`\Sk`（尖点形式空间）等。
- 交叉引用一律用 `\cref{}`（由 cleveref 提供）。

### 章节结构
- 每章以简短导言开头，说明本章目标与主要结果。
- 核心内容按 section 组织，定理-证明穿插例子和注记。
- 每章末尾有习题节（`\section{习题}`），每道习题后附参考答案。
- 记号表（longtable）放在 `main.tex` 的目录之后、正文之前，作为全书查阅参考。
- 章内的 TODO 注释（`% TODO:`）标记待填充内容，写完后删除。

### 语言风格
- 中文数学写作风格，专业但易读。
- 术语首次出现时给出中英文对照，如"尖点形式（cusp form）"。
- 证明结尾用 `\qedhere` 或让 amsthm 自动添加 QED 符号。

## 工作流程

每次写完一章（或对章节做了修改）后，必须执行以下步骤：

1. **编译 PDF**：运行两遍 `xelatex` 确保目录和交叉引用正确：
   ```bash
   cd /home/szylover/chinese-modular-forms
   xelatex -interaction=nonstopmode -halt-on-error main.tex
   xelatex -interaction=nonstopmode -halt-on-error main.tex
   ```
2. **确认无错误**：检查编译输出无 `!` 错误。
3. **提交并推送**（含 PDF）：
   ```bash
   git add -A && git commit -m "描述本次修改" && git push
   ```

PDF 文件 `main.pdf` 已纳入版本管理，每次提交都应包含最新编译的 PDF。
