\section{提交前 90 秒检查}

\begin{enumerate}
  \item \textbf{空输入}：\code{null}、空数组、单节点、容量 0 是否处理？
  \item \textbf{整数}：路径和、乘积、平均数是否需要 \code{long}？
  \item \textbf{Comparator}：是否用了 \code{Integer.compare} 而不是相减？
  \item \textbf{堆方向}：Java 默认小顶堆；大顶堆是否传了 \code{Comparator.reverseOrder()}？
  \item \textbf{队列}：是否使用 \code{ArrayDeque}，且没有放 \code{null}？
  \item \textbf{状态重置}：成员变量 \code{answer}、\code{visited} 是否在 public 方法开头重置？
\end{enumerate}

\section{性能不降级的三个动作}

\begin{enumerate}
  \item 频繁索引的数值状态用 \code{int[]}，不要无脑 \code{List<Integer>}。
  \item 字符串构造用 \code{StringBuilder}。
  \item 需要容量时预分配：\code{new ArrayList<>(expectedSize)}，但不要为了微优化牺牲可读性。
\end{enumerate}

\begin{pitfall}[递归深度]
Java 栈通常比 C++ 算法环境更容易在极深 DFS 时溢出。普通二叉树题可递归；链状图、十万节点树、\code{number-of-islands} 的极端输入，更安全的是显式 \code{ArrayDeque}。
\end{pitfall}
