# 《中国大事编年》书系 Agent 生产蓝图

## 开始前的硬规则

1. 先读 `SERIES_SPEC.md`，再读本文件和目标卷 `volumes/<volume>/AGENTS.md`。
2. 不在 `main` 分支工作。先运行
   `scripts/new-agent-worktree.ps1 -TaskId <id>`，只在生成的
   `chronicle/<task-id>` worktree 中写入。
3. 只能认领 `WORK_QUEUE.yaml` 中状态为 `ready`、且范围明确锁定的任务。
   认领后立即写入 `in_progress`、分支和开始时间；一个文件只能有一个
   writer。
4. 不跨任务范围修改文件。`main.tex`、chapter wrapper、全局索引和 PDF
   只由 Integrator 修改；正文、账本、来源和图件分别由已锁定的角色负责。
5. 事件先进入账本，Narrative 只使用已有账本 ID；Map 只写独立图文件。
6. 修改本文件或卷内 `AGENTS.md` 后，必须先交用户审阅。用户明确确认前，
   不得开始或继续受该规范影响的账本、正文、地图、构建、发布或内容任务。

## 卷的共同产物

每卷必须形成可核验的生产链：

```text
史料池 -> 事件账本 -> 连续叙事 -> 专题/图件 -> 附录索引 -> 可复现构建 -> 发布
```

- `main.tex` 只包含卷面、目录、分部和 chapter wrapper 的顺序，不堆积正文。
- 正文按连续的事件簇拆为独立文件；国家或政权存入 `states/<polity>/`，
  共享年序与慢变量存入 `common/`，跨政权问题存入 `synthesis/`。
- 每卷都要有来源附录、事件/政权/主题/来源/图件索引、书系承接说明和
  `PROGRESS.md`。卷内 `AGENTS.md` 只补充范围、账本、材料、目录和争议的
  本卷例外，不得弱化本文件。
- 每卷开篇必须有简短的“怎样读这一卷”方法说明，解释本卷的时间边界、
  年序材料、主要证据限制、因果读法与信息框用法；不能假定读者已读首卷。

## 编年体是正文主线

- 本书系是编年体通俗史：正文首先按时代、年号/年份、政权更替与连续事件簇
  前进，让读者始终知道“何时发生、先后怎样相承、压力如何累积和转移”。
- 国家 dossier、制度、经济社会、文化思想、边疆和跨政权专题用来解释这条
  年序主线，不能取代它，也不能把全卷写成按主题排列的历史百科。
- 每个大时期必须有共享年序；国家 dossier 以该政权的时间进程组织。专题应
  从已建立的事件链进入，并回到具体事件的前因、后果和后续时间位置。
- 事件账本服务编年覆盖与因果核验，但正文不能逐条转录账本。连续叙事应在
  保持时间推进的同时，为制度、社会和文化的慢变量留出必要的专题段落。

## 编年覆盖与章节节奏

- 每卷先建立按时间带、政权/地区和事件类型交叉的覆盖表，并在来源矩阵中
  显示材料密度与空带。先补 C 级桥接事件，再补改变联盟、继承、征发、资源
  归属或区域控制的 A/B 级节点；不得以名场面替代连续年序。
- 账本和 `PROGRESS.md` 必须列出未覆盖年代、被大国叙事遮住的小国/地方、
  慢变量和待核争议。材料稀薄处应留下空白并说明其来源，而非用后出故事伪造
  逐年细节。
- 每个大时期依次组织为：时代纲要、共享年序、国家/政权时间线、跨政权专题、
  经济社会与文化思想、章节纲要。可因材料调整合并，但不得取消共享年序或以
  专题开篇替代时间推进。
- 国家平衡按行动者而非名气衡量。小国、地方、迁徙、城邑、道路、继承和受
  战争影响的人群不能只是大国兴亡的布景；若材料不足，应明说不可见性。

## 事件账本与正文接入

账本是覆盖率的唯一检查表。每条记录至少包含：

```text
internal_id, date/reign_year, polity, event, confidence,
core_sources, source_caveat, modern_research_direction,
causes, consequences, target_file, tier
```

- A 级是改变政治、制度、战争、经济或文化走向的转折，必须多页深描；
  B 级是国家线不可缺少的完整条目；C 级至少保持年序和因果连续。
- 正文展示中文可点击事件名；内部 ID 仅出现在账本、锚点和维护文件。
- 每个事件簇必须交代时间、空间、行动者、资源与选择、结构性原因和触发、
  直接后果与中期影响，并链接可核验的前因或后果。
- 不把时间相邻写成自动因果；明确区分结构性原因、触发因素、使能条件、
  直接后果、中期影响与反馈回路。
- 账本中的拆分锚点可服务核验，但不得被逐条模板化改写成主体 prose。
- C 级节点可以短写或仅承担桥接，但不得套用“事件—前因—来源—研究—后果”
  的固定段落。每段的长度和证据密度必须由材料及其叙事作用决定。
- 事件首次深描处必须设置中文可点击锚点；国家章、专题、图件说明和索引均
  回链同一锚点，避免同一事件在不同文件重复展开。

## 历史内容的六层与章节组织

每卷同时处理：

1. 编年政治、继承、地方权力与制度；
2. 土地、税收、货币、贸易、劳役和财政；
3. 社会结构、家族、人口、迁徙、性别与流动；
4. 军事、边疆、道路、技术、环境与地理；
5. 文化、教育、礼制、宗教、文学与思想；
6. 史料、叙事立场与解释争议。

每个国家/政权 dossier 开头应给出地理、资源、人口、制度、军政、精英、
经济机制与继承难题的纲要。跨国人物、制度、战争、外交、迁徙和思想流动
另写合卷，回链各政权事件；不得把同一问题机械切碎。大事件簇结尾必须写
“制度得失”：当时解决什么、成本由谁承担、产生什么副作用、后续何事承接
该压力。各章以纲要收束细节并接到下一阶段。

## 来源、争议与叙事

- 先依事件问题选择材料：正史和编年材料维持年序，志书和文书处理制度，
  列传与同时代文本处理人物，考古、墓志、宗教和地方材料处理物质与区域。
  《资治通鉴》及续编是编年脊柱，不替代对应时代的正史与专门材料。
- 不建立死板来源等级，也不把官修叙事、诏令、数字、族名、疆界、政策条文
  或个人动机写成超过材料承载力的事实。对不一致处说明材料层次、可说范围
  与未决争议。
- 每卷在来源矩阵中定义适用的材料标记，例如“经年、叙事、后设、物证、
  解释”。每一深描事件至少写出最小可证的日期/顺序和主体、一条基础材料及
  其层级、一项可核的现代研究方向，并说明关键细节不能证实到哪里。
- 兵数、伤亡、路线、盟辞、制度执行、人口财政数字和人物动机均是高风险
  声明。只有材料明确支持时才写为事实；否则标明“史书称”“传世叙事称”或
  “研究推测”，不得借叙事流畅抹去限度。
- 叙事从可见的诏命、道路、城邑、河流、器物、调兵或战场进入，再展开资源、
  制度与社会条件。人物是受信息、联盟、家族和制度约束的行动者。
- 会盟、救援、远征、迁徙和战争必须落实到城邑、关口、渡口、道路、粮秣、
  安置或可通行性等空间条件；不能只用政权名和箭头叙述行动。
- 禁止虚构对话、内心、服饰或无材料支撑的场景；推断必须与确证事实分开。
  古文窗口仅保留短引文、出处、语境和立场，不作逐句翻译或现场实录。

## 文学性编辑门槛

历史可靠不等于读来有文学性。每个 A 级事件和每个国家/政权 dossier 必须在
入书前通过以下编辑检查：

1. 开头让读者看见一件由材料支持的事物、地点、命令、道路、城门、河流、
   队伍或文书，再自然进入年份与背景；不得用百科式定义或结论先行开场。
2. 叙事中必须有行动者在资源、信息、联盟、家族或制度约束下的选择，并写出
   选择的代价和对后来行动的压力；不以“明君/昏君”“忠臣/奸臣”等标签代替。
3. 段落要推动行动、处境或读法，不能把账本字段改写成连续的说明书句子；
   避免反复使用“这说明”“重要的是”“由此可见”等空泛总结。
4. 长句用于铺陈关系和空间，短句用于转折与结果；保留叙事留白，不用虚构的
   对白、心理、衣饰或戏剧化细节填满材料沉默。
5. 引文、信息框和制度解释必须改变读者对眼前事件的理解；若删去后正文无损，
   则删去或改入脚注/附录，不把它们当作知识展示。

Narrative 完成后，Integrator 或独立 Continuity Editor 必须抽读每个事件簇的
开头、转折和结尾；任何一处若删去专名、年份后仍可套用于多数历史事件，即视为
模板 prose，退回重写。文学性来自受约束的行动、空间和节奏，不来自堆砌形容词
或仿古腔。

不得以“深描扩写”“补叙”或类似批量任务，把同一组抽象段落替换少量专名后
重复用于不同事件。提交前必须检查锁定范围内的跨事件重复文本；引文、固定方法
说明和索引条目除外，任何可独立成段的重复 prose 都须改写为该事件特有的材料、
空间、行动者和后果。发现批量模板时，先删除或重写模板段，再谈增加覆盖数量。

## 基础书目与读法索引

每卷的来源附录除事件来源路径外，必须有读者可检索的“基础书目与读法索引”。
每项书目写明书名、作者/编者、所用版本、适用时段或问题、在本卷中的用途和
不能承担的判断；不得罗列未实际使用的书名来制造权威。

- 书系的正史骨架必须完整列出二十四史：`《史记》《汉书》《后汉书》《三国志》
  《晋书》《宋书》《南齐书》《梁书》《陈书》《魏书》《北齐书》《周书》
  《隋书》《南史》《北史》《旧唐书》《新唐书》《旧五代史》《新五代史》
  《宋史》《辽史》《金史》《元史》《明史》`。各卷只重点使用与本卷有关的
  史书，但要在书系索引中说明覆盖卷次和材料局限；清代另以《清实录》、
  《清史稿》、档案和地方文献接续，不能假装属于二十四史。
- 《资治通鉴》及续编用于建立跨政权年序和追踪事件链，不替代相关正史、
  实录、国别史、文书或出土材料。宋、明、清等卷须在索引中列出各自不可省略
  的专门编年与档案材料。
- 《国史大纲》《历代政治得失》《中国史纲》及吕思勉的通史/断代著作属于
  解释性读物：分别标明它们用于时代大势、制度得失、问题意识或社会经济
  长线，不能充当逐年事实、精确数字或人物动机的唯一来源。
- 《古文观止》是古文阅读的选本，不是独立史料。使用其中篇章时必须回溯至
  原始文本、可靠底本和成文语境；索引可把它列为阅读入口，不能把选本的篇名
  或注释当成原始证据。
- 古文窗口、事件注记和书目索引应互相链接：读者能从一段引文回到原典及其
  材料层级，也能从书目知道该书在何卷、何类问题中被怎样谨慎使用。

## 图件、索引与排版

- 图表、时间轴、路线和地图一律为 `figures/<period>/` 的独立、可版本化
  TikZ 文件，正文只 `\input`。不得嵌入正文或搬运许可不明的网络地图。
- 使用暖纸色、朱褐和铜色。图件必须写明“示意”或“疆界约略”，标出材料
  限度和空间关系，不伪造精确国界。
- 每个大时期至少有一张空间总览和一条简洁时间轴；每个国家/政权 dossier
  至少有位置或关系图；大战、迁徙、道路、运河/贸易路线和边疆变化按需要
  增设局部图。
- Indexer 只索引实际存在的正文锚点和图件；账本保留全部覆盖记录，不能因
  未被索引或未深描而删除。

## 任务型信息框

信息框是阅读中的“停靠点”，用来把关键事件、材料边界、原文或因果关系从
连续叙事中清楚地提出；它们不是装饰，不得替代正文，也不得每页堆砌。

- `event` 用于首次深描的编年锚点。标题行应给出可读的事件名、日期/王年、
  政权或地点、以及必要的年代限度；框内把读者带入该事件的连续叙事。
- `turningpoint` 只用于改变后续政治、制度、战争或社会路径的节点，标题要
  点明具体转折，而非泛写“重要事件”。
- `causalchain` 用于把结构性原因、触发因素、使能条件、直接后果、中长期
  影响或反馈回路拆清；可用简短表格帮助读者，但不能把因果压成一句空话。
- `textwindow` 用于短原文。必须提供出处、成文语境、文本立场，以及它能照亮
  什么、不能证明什么；原文不是现场实录或权威结论的装饰。
- `sourcenote` 与 `debate` 用于“材料能证实到哪里”和会改变读法的史学分歧：
  分开近时证据、后出叙事、官方修史和现代解释，明确可说范围与未决问题。
- 统一沿用暖纸底、朱褐强调线和深褐标题带的层级；标题写成读者需要解决的
  问题，例如“《尚书·牧誓》不是战地录音”“牧野能证实到哪里”，不用无信息
  的“补充”“资料”或“说明”。
- 每卷 preamble 必须提供兼容的 `event`、`turningpoint`、`causalchain`、
  `textwindow`、`sourcenote` 与 `debate` 环境；优先复用
  `shared/styles/` 的书系样式。不得为图方便以普通引用块、颜色各异的卡片或
  正文内嵌 TikZ 替换这些约定。

## 卷一蓝图验收

在声称一个卷具有“卷一蓝图”前，Integrator 必须逐项确认：

1. 有面向读者的方法说明、时代纲要、共享年序、国家/政权时间线、专题与章节纲要；
2. 账本、来源矩阵和覆盖表可显示年代空带、行动者失衡与材料限度；
3. 深描事件有中文锚点、最小证据链、前因后果链接，且正文没有账本模板化痕迹；
4. 国家、地方、小国、边疆、迁徙与慢变量不会被单一大国主线抹去；
5. 信息框依任务使用，并明确原文、证据、争议与因果各自能说明的范围；
6. 图件独立、可版本化，含时代总览、dossier 位置/关系图及必要的局部空间图；
7. 附录索引只链接已写入正文的事件和图件，完整账本仍单独保留；
8. 构建、链接、引用、重复锚点、进度、PDF 路径与发布同步均由 Integrator 验收。

## 角色、交接与验收

| 角色 | 允许写入 | 交付要求 |
|---|---|---|
| Chronology | `data/event-ledger/` | 来源、层级、前因后果与目标文件完整 |
| Source | `data/source-corpus/`、证据注记 | 材料边界、争议和可用路径明确 |
| Narrative | 任务锁定的正文 `.tex` | 仅扩写已入账事件，保持连续叙事 |
| Cartographer | `figures/` | 独立示意图、图注和材料限度 |
| Indexer | 任务锁定的附录/索引 | 只链接已存在的锚点、来源和图件 |
| Integrator | wrapper、`main.tex`、进度、构建与发布 | 合并前验链接、引用、锚点、路径和队列 |

完成任务时报告：修改文件、事件 ID、核心来源、争议、构建结果与未完成事项；
同步更新本卷 `PROGRESS.md` 和队列状态。

仅 Integrator 可以合并到 `main` 或发布。PDF 只能位于仓库根
`pdf/china-chronicle/`，命名为 `volXX-<slug>.pdf`；每次合并并更新书系
PDF 后，Integrator 必须运行 `scripts/publish-downloads.ps1`，同步整个目录到
Azure Blob 账户 `szydownloads` 的
`downloads/books/china-chronicle/`，刷新下载目录页文件大小并确认直接 Blob
链接可访问。
