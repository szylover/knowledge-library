# 《中国大事编年》写作指令

## 目标

写一部从周到清末的长篇通俗编年史。它必须有故事感、人物和权力关系，也必须经得起史料和制度分析；不是皇帝年表，不是纯政治教材，也不是虚构历史小说。

第一长卷是西周至秦。它将确立全书的事件密度、叙事语气、史料标注和因果链标准。

## 文件结构

- `main.tex` 只放封面、目录、分部与 `\include` 顺序，绝不堆积正文。
- 每个时代是独立章节 wrapper，例如 `chapters/ch03-spring-autumn.tex`。
- 大时期使用 `common/` 保存共享时间线，使用 `states/<state>/` 保存国家/政权 dossier。
- 事件簇按问题、在位期或连续年份拆分为独立 `.tex` 文件。单个文件过长时必须继续拆分。
- 跨国人物、制度、战争、外交、迁徙、思想流动另写 `synthesis/` 合卷，不在各国章节中机械重复。

## 总体史六层

任何时代都要同时看：

1. 年份与政治事件；
2. 国家制度、官僚、继承与地方权力；
3. 土地、税收、货币、贸易、劳役与财政；
4. 社会结构、家族、人口、迁徙、性别与流动；
5. 军事、边疆、地理、技术和环境；
6. 文化、教育、礼制、宗教、文学与思想。

政治编年是导航脊柱，不是全书内容。慢变量应写进 `经济与社会`、`文化与思想` 子章，并与事件 ID 相互链接。

## 事件标准

每个可靠纪年的重要事件必须单列，并使用稳定的内部事件锚点，例如 `WZ-0841-GONGHE`。内部锚点不得直接展示给读者；正文只显示中文可点击事件名，例如 `〔西周·共和〕`、`〔春秋·城濮之战〕`。

事件条目依次回答：

1. 日期、地点、行动者、年代可信度与事件类型；
2. 这一年发生了什么；
3. 事情为什么走到这里：结构性原因、近期触发、联盟和可用资源；
4. 当事人面对哪些选择、做了什么、付出什么代价；
5. 后来改变了什么：直接结果、中期影响和明确的后续事件 ID；
6. 史料如何说、哪里存在争议。

所有前因、后果和跨国引用都必须使用中文可点击事件链接，跳转到对应事件条目；不要在正文暴露英文、拼音或数字工程编号。

不要把时间相邻自动写成因果。因果关系使用“结构性原因、触发因素、使能条件、直接后果、中期影响、反馈回路”等明确标签。

## 国家分章与跨国合卷

- 春秋必须分写周王室、齐、晋、楚、秦、鲁、郑宋、吴越，同时保留共享年表。
- 战国必须分写秦、楚、齐、燕、韩、赵、魏与残余周室，同时保留共享年表。
- 一人或一项制度跨越多国时写合卷：例如李悝、吴起、申不害、商鞅进入“变法与国家能力”专题，而不是被切碎在魏、楚、韩、秦四处。
- 合卷必须回链各国事件 ID，说明共同结构与不同结果，不能脱离国家章节凭空概论。

## 纲要与制度得失

- 每朝、每国 dossier 开头写 `纲要`：地理、资源、人口、制度、军政、精英、经济机制和继承的问题。
- 每个大事件簇结尾写 `制度得失`：它在当时解决了什么？谁承担成本？制造了什么副作用？后续哪一事件继承了这个压力？
- 得失不是道德判词。说明衡量标准：国家能力、财政可持续性、军事实力、社会流动、地方自治、文化整合等。
- 每章末尾写 `本章纲要`，把细节重新收束为历史长线和下一章继承的具体问题。

## 史料与争议

- 周代重点使用《尚书》《春秋》《左传》《国语》《战国策》《史记》、竹书纪年、金文与简帛研究。
- 后续卷以二十四史、《资治通鉴》及续编、实录、地方文书、出土材料和现代研究交叉使用。
- 每条深描事件至少说明一条基础史料与一项现代研究方向；没有核实的细节不能写成确定事实。
- 区分同时代材料、后出叙事、官方修史与现代解释。年代或动机有争议时直接写出争议，不伪造精确性。

## 古文窗口

- 在它真正照亮事件时插入《左传》《国语》《战国策》《论语》、太史公曰、《过秦论》等短原文。
- 通俗且知名的古文不逐句翻译；说明作者、成文处境、对应事件、文体与立场即可。
- 仅对生僻词、关键典故或存在分歧的概念作短注。
- 古文不是装饰，也不是自动正确的证词。比如《过秦论》是汉人回望秦亡的政治论说，不能当作秦朝现场记录。

## 叙事风格

- 以一场危机、一次选择、一年中的转折、一个奏章、一次远行或一处战场为事件簇入口，再向外展开制度和社会背景。
- 人物是受信息、资源、家族、联盟和制度约束的行动者；写选择和代价，不贴“明君”“昏君”“忠臣”“奸臣”式标签。
- 可借鉴微观切口、制度纵深和严肃历史剧的权力关系组织，但全部使用原创叙述。
- 禁止虚构对话、内心独白、服饰细节或未见于材料的戏剧场景。
- 推断用“从现存材料看”“很可能”“一种解释是”等措辞，与确证事实分开。
- 面向普通读者，先讲人能看见的事情，再解释制度；避免科学教材口吻、术语堆砌和流水账。

## 视觉与排版

- 使用暖纸色、朱褐和铜色，不使用冷蓝科技风。
- 每个大章节开头有简洁时间轴；时间轴服务阅读，不堆砌数据。
- 地图是第二条叙事线。每个时代 chapter 必须有一张总览图；每个国家 dossier 要有位置图；大战、东迁、迁徙、运河/贸易路线和边疆变化要有局部图。
- 地图一律用可版本化的 TikZ 自绘，放在 `maps/` 子目录并由章节 `\input`。不得直接搬运无明确许可的网络地图。
- 每张图写清“示意图”或“疆界约略”，不能把材料不足时代的边界画成现代地图般精确；图注说明该图要帮助读者理解的空间关系与主要参考材料。
- 地图优先标出都城、河流、山口、交通线、战场、封国/国家相对位置和变化方向；颜色用于区分政治体和路线，不用于制造虚假的确定性。
- 使用 `event`、`turningpoint`、`causalchain`、`sourcenote`、`debate`、`text-window` 等环境，但正文仍应像一本可连续阅读的书。
- 不要把每页都装成彩色卡片；关键节点才使用视觉框，普通叙事保持流动。
